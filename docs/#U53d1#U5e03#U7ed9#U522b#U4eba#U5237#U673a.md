# 发布给别人刷机

这份说明是给维护者看的，不是给最终用户看的。

目标是把一次本地编译后的固件，打包成一个可以直接发给别人使用的压缩包。别人不需要安装 ESP-IDF，只需要拿到你打好的发布包即可。

## 一次完整流程

1. 启动后端，确认 `/admin` 和 `/dashboard` 正常。
2. 编译固件：

```bash
task build
```

3. 打发布包：

```bash
python tools/package_release.py --version v2026.04.15
```

4. 打包结果会出现在：

```text
release_dist/m5paper-dashboard-v2026.04.15/
```

## 也可以直接用 GitHub 编译

仓库已经包含 GitHub Actions 工作流：

- 普通 `push` / `pull request`：自动跑后端检查并编译固件
- 推送标签 `v1.0.0`：自动编译固件、打发布包，并把 zip 挂到 GitHub Release

如果你不想在本地装 ESP-IDF，可以把代码推到 GitHub 后，在 Actions 页面下载 artifact，或者直接从 Release 页面下载打好的发布包。

## 发布包包含什么

- `flash_windows.bat`
  - 给 Windows 用户的一键烧录入口
- `firmware/*.bin`
  - 实际烧录用的 bootloader / partition-table / app 固件
- `flash_manifest.json`
  - 给 ESP Web Tools 或静态网页烧录使用
- `README_先看我.txt`
  - 发给用户时让他先看的中文说明

## 给用户怎么说

你可以把整个发布包压缩后发给用户，然后告诉他：

1. USB 连接设备
2. 双击 `flash_windows.bat`
3. 输入串口号
4. 烧录完成后，设备第一次会自动开启热点 `M5Paper-Setup`
5. 手机访问 `192.168.4.1` 填 Wi-Fi 和 API_URL

## 注意事项

- 如果设备已经深睡，用户可能需要先按住 `Boot`，再点按 `Reset`，然后立即开始烧录。
- `flash_windows.bat` 不要求 ESP-IDF，但仍然依赖本机存在 Python 3；脚本会自动安装 `esptool`。
- 如果你要进一步做到“完全零安装”，下一步建议把 `flash_manifest.json` 放到静态网页上，配合 ESP Web Tools 做网页烧录。
